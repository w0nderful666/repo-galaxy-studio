# Module Matrix

标记说明：

- **必须**：该等级项目应默认包含。
- **推荐**：通常有价值，但可按项目规模裁剪。
- **可选**：只有在业务需要时加入。
- **不建议**：该等级阶段容易过度设计，默认不要加入。

| 模块 | C 级 | B 级 | A 级 | 说明 |
| --- | --- | --- | --- | --- |
| 现代首页 Hero | 必须 | 必须 | 必须 | 用第一屏明确项目是什么、适合谁、隐私原则是什么。 |
| 工具卡片 | 必须 | 必须 | 必须 | 展示工具入口或模板能力，按钮必须真实可用。 |
| 深色模式 | 必须 | 必须 | 必须 | 使用 CSS variables 或等价方案，刷新后保持设置。 |
| 中英文切换 | 必须 | 必须 | 必须 | UI 主路径文案必须完整，不混用半成品文案。 |
| localStorage | 推荐 | 必须 | 必须 | C 级可只保存主题语言；B/A 级必须封装并复用。 |
| 导入 JSON | 可选 | 必须 | 必须 | B/A 级用于配置、任务或数据恢复。 |
| 导出 JSON | 可选 | 必须 | 必须 | 用于备份、分享配置或保存处理结果。 |
| 分享链接 | 可选 | 必须 | 必须 | B/A 级用于分享当前状态或参数。 |
| 文件拖拽 | 可选 | 推荐 | 必须 | 文件工具建议支持；A 级工具箱应统一体验。 |

| 空状态 | 推荐 | 必须 | 必须 | 不让用户面对空白页面。 |

| 最近使用 | 可选 | 推荐 | 必须 | A 级多工具场景中非常重要。 |

| README 模板 | 必须 | 必须 | 必须 | 必须说明用途、运行、部署、隐私和自测。 |
| RELEASE_NOTES | 可选 | 必须 | 必须 | B/A 级每次发布都应记录变更。 |
| SEO / OpenGraph | 推荐 | 推荐 | 必须 | A 级项目应完整支持分享预览。 |
| 隐私说明 | 推荐 | 必须 | 必须 | 明确本地处理、不上传文件、不追踪。 |
| GitHub Pages 部署 | 必须 | 必须 | 必须 | 所有等级默认支持 GitHub Pages。 |

## 使用方式

创建新项目时，先确定等级，再按矩阵裁剪模块。不要因为 A 级存在某个模块，就把它提前塞进 C 级项目。

升级项目时，优先补齐当前等级的“必须”项，再考虑“推荐”项。
