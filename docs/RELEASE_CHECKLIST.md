# Release Checklist

发布前逐项检查。任何“必须”项不通过，都不应发布。

## 功能真实性

- [ ] 所有按钮是否真实可用。
- [ ] 是否存在空壳 UI。
- [ ] 示例数据是否能真实触发功能。
- [ ] 一键复制是否能复制预期文本。
- [ ] 下载按钮是否能生成真实文件。
- [ ] 弹窗、关闭、切换等交互是否可恢复。

## 代码状态

- [ ] 是否存在 TODO/FIXME。
- [ ] 是否存在调试输出。
- [ ] 是否存在未使用的大段代码。
- [ ] 是否引入了不必要的重依赖。
- [ ] `npm run build` 是否通过。
- [ ] self-test 是否通过。

## 体验与可访问性

- [ ] 移动端是否正常。
- [ ] 桌面端是否正常。
- [ ] 深色模式是否正常。
- [ ] 中英文是否完整。
- [ ] 文本是否溢出按钮或卡片。
- [ ] 空状态是否清楚。
- [ ] 错误状态是否清楚。
- [ ] Toast 或等价反馈是否可见。

## 文档

- [ ] README 是否完整。
- [ ] RELEASE_NOTES 是否完整。
- [ ] 项目等级说明是否准确。
- [ ] GitHub Pages 部署路径是否正确。
- [ ] 隐私说明是否清楚。
- [ ] 自测方法是否清楚。

## 隐私与安全

- [ ] 是否错误上传用户文件。
- [ ] 是否依赖后端。
- [ ] 是否存在外部追踪脚本。
- [ ] 是否存在敏感信息。
- [ ] 是否暴露 token、密钥、cookie 或本地绝对路径。
- [ ] 是否使用不必要的外部 CDN。

## GitHub Pages

- [ ] Vite `base` 是否适配仓库路径。
- [ ] 构建产物是否能从相对路径加载资源。
- [ ] `self-test.html` 是否包含在构建产物中。
- [ ] 部署说明是否与当前仓库一致。

- [ ] 可以发布。
